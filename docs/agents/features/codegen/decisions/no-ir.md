# Decision: No Intermediate Representation File

## Context

Code generation requires structural information about the model (state names,
transition tables, attribute types, resource capacities). This information must
flow from the Rust model definitions to the codegen backends. The options
considered were:

1. **IR file**: the proc macros emit a serialized representation (e.g., JSON)
   that the codegen tool reads.
2. **No IR file**: the codegen binary imports the model crate as a Rust
   dependency and reads metadata directly from trait impls generated by the proc
   macros.

## Decision

No IR file. The codegen binary reads the model directly from Rust trait impls.

## Rationale

### The IR would duplicate information already in the Rust types

The proc macros generate trait impls that carry all structural metadata (state
names, transitions, capacity types, usage bindings). Serializing this to JSON
and then deserializing it in the codegen tool adds a serialization boundary
that can have bugs, a schema to version and validate, and an artifact to keep
in sync, all for information that is already available in compiled form.

### The codegen tool is Rust

Since the codegen binary is a Rust program in this repository, it can depend on
the model crate directly. No need for a language-neutral exchange format.

### Adding target languages does not require an IR

Each target language (C++, Java, Python) is a backend in the same Rust codegen
binary. They all read from the same in-memory model representation. Adding a
target means writing a new emitter function, not a new IR consumer.

### Escape hatch exists

If a future requirement demands codegen tooling written in a non-Rust language,
`Serialize` can be derived on the in-memory model types to produce JSON. This is
a one-function addition, not an architectural change. The internal types already
exist; serialization is opt-in.

### What consumers need and how they get it

| Consumer                  | When         | Source                              |
|---------------------------|--------------|-------------------------------------|
| Rust instrumentation API  | compile time | proc macro generates directly       |
| C++/other codegen backend | build time   | codegen binary imports model crate  |
| Analyzer                  | runtime      | model registration events           |
| UI                        | runtime      | analyzer serves model metadata      |

No consumer requires a serialized IR file.
