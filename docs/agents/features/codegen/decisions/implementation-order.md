# Decision: Implementation Order

## Order

1. **`quent-model` core types** — `Ref<T>`, `Usage<T>`, `Resource` trait,
   `ModelComponent` trait, `Model<T>`, `FsmEvent<S, D>`. No proc macros yet,
   just the type foundations.

2. **`quent-model` proc macros** — `#[derive(Fsm)]`, `#[derive(State)]`,
   `#[resource(...)]`, `#[derive(Entity)]`, `#[event]`,
   `#[deferred]`, `#[usage]`, `#[derive(ResourceGroup)]`.
   Start with `#[derive(Fsm)]` + `#[derive(State)]` as the core.

3. **`quent-stdlib`** — Memory, Processor, Channel FSM definitions using the
   proc macros. Validates that the macros produce correct code for real
   definitions.

4. **Rewrite simulator model** — Define the simulator's Task FSM and resources
   using the new annotations. Validates cross-crate composition, model
   collection, and the full Rust instrumentation API.

5. **Extensive Rust codegen tests** — Verify all generated code before adding
   FFI complexity:
   - Generated event enums (transition variants, deferred variants)
   - Trait impls (Entity, Fsm, FsmTypeDeclaration, Resource, ModelComponent)
   - `FsmEvent<S, D>` with sequence numbering
   - Deferred fields: state handle borrowing, setter emission, merge semantics
   - `Usage<T>` resolution from FSM operating state to capacity type
   - `Ref<T>` compile-time validation
   - Model collection via `Model<T>` tuple recursion
   - Drop behavior (auto-exit)
   - Cross-crate composition (domain model + application model)

6. **`quent-codegen` with CXX bridge backend** — The codegen binary that
   imports a model crate and emits `#[cxx::bridge]` modules. C++ headers
   generated by CXX.

7. **Migrate Sirius** — Replace hand-written CXX bridges and event types with
   generated ones. Real-world validation.

## Rationale

Each step validates the previous one. The proc macros are tested first with
standard library definitions (step 3), then with a real application model
(step 4), then exhaustively (step 5) before any FFI code generation is
attempted. This ensures the Rust foundation is solid before adding the
complexity of cross-language code generation.
